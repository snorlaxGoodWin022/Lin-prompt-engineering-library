1. **识别可以懒加载的模块**
   - 按路由拆分
   - 按功能模块拆分
   - 第三方库动态导入

2. **提供路由懒加载代码**
   - React lazy + Suspense 示例
   - Vue 异步组件示例
   - 加载状态处理

3. **第三方库按需引入方案**
   - UI 库按需导入配置
   - 图表库按需加载
   - 工具库 tree shaking 配置

4. **提取公共 chunks 配置**
   - vendor chunk 配置
   - runtime chunk 配置
   - 缓存优化策略

5. **预计优化后体积**
   - 主包大小
   - 异步 chunks 分布
   - 总体积变化

## 模板说明

此模板用于分析打包产物并生成代码分割方案。通过合理的代码分割，可以显著减少首屏加载时间，提升用户体验。

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `{Webpack/Vite/Rollup}` | 打包工具类型 | `Webpack` |
| `{xxx MB}` | 主包大小 | `2.5 MB` |
| `{列出体积大的库}` | package.json 中大型依赖 | `lodash, moment, echarts` |
| `{大约多少个页面}` | 路由数量 | `约30个页面` |
| `{是/否}` | 是否需要 SEO | `是` |

### 适用场景

- 首屏加载时间过长需要优化
- 打包产物体积过大需要分析
- 需要实施代码分割策略
- 第三方库体积优化

### 使用建议

1. 先运行 `webpack-bundle-analyzer` 生成分析报告，填入关键数据
2. 列出体积最大的 5-10 个依赖库
3. 说明项目的路由数量和首屏加载要求
4. AI 会根据打包工具生成对应配置代码

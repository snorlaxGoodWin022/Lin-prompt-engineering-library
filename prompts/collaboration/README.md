# 团队协作模板

用于规范团队协作流程的 Prompt 模板。

## 可用模板

### Git 工作流
- **git-commit-messages.md** - 生成符合规范的 Git Commit Message
- **pr-descriptions.md** - 编写详细的 Pull Request 描述

### 文档生成
- **technical-documentation.md** - 快速撰写技术文档（设计文档、接口文档等）

## 使用场景

1. **提交代码**：生成规范的 Commit Message，方便代码追溯
2. **代码审查**：编写清晰的 PR 描述，提高 Review 效率
3. **文档维护**：快速生成技术文档，保持文档与代码同步
4. **新人培训**：统一团队协作规范，降低沟通成本

## 模板特点

- **规范统一**：遵循团队约定的格式和标准
- **内容全面**：包含必要的上下文信息
- **节省时间**：自动化重复性文档工作
- **质量保证**：确保关键信息不遗漏

## 最佳实践

1. 根据团队实际规范调整模板格式
2. 提交前确认 Commit Message 准确反映改动内容
3. PR 描述中说明测试情况和注意事项
4. 技术文档需定期更新以保持时效性

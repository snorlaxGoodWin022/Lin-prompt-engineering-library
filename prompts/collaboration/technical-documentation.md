## 文档结构要求

### 通用结构
1. **标题和概述** - 文档标题和简要说明
2. **目录** - 自动生成的文档目录
3. **背景和目的** - 为什么需要这份文档
4. **正文内容** - 根据文档类型组织
5. **总结和后续** - 关键点和后续计划
6. **附录** - 参考资料、术语表等

### 按文档类型细化
- **功能设计文档**：背景 → 需求分析 → 方案对比 → 最终方案 → 实施计划 → 风险评估
- **接口文档**：接口列表 → 请求参数 → 响应示例 → 错误码 → 调用示例 → 版本变更
- **组件库文档**：组件介绍 → 安装使用 → API文档 → 示例展示 → 最佳实践 → 常见问题
- **项目架构文档**：架构概述 → 技术选型 → 模块设计 → 数据流 → 部署结构 → 扩展计划

## 内容要求

1. **结构清晰**，有完整的目录结构
2. **用 Markdown 格式**，便于维护和阅读
3. **代码示例带注释**，说明关键部分
4. **有流程图的地方用 Mermaid**，可视化复杂流程
5. **关键点用表格整理**，便于对比和查阅
6. **术语统一**，保持一致性

## 输出要求

- 目标读者是 `{目标读者}`，用他们能看懂的语言写
- 保持专业性和准确性
- 提供足够的细节，但避免过度冗长
- 考虑文档的维护和更新成本

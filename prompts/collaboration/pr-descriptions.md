# Pull Request 描述生成模板

帮我写一个 Pull Request 描述。

## 改动内容

```
{简要说明这个PR做了什么，解决了什么问题}
```

## 关联 Issue

```
{如果有的话，贴issue链接或编号}
```

## 测试情况

- **单元测试**：{通过/未通过/未写}
- **集成测试**：{通过/未通过/未写}
- **手动测试场景**：{列出你测了哪些功能}
- **浏览器兼容性**：{测试了哪些浏览器}

## PR 描述要求

请生成完整的 PR 描述，包含以下部分：

### 1. 改动说明
详细说明这个 PR 做了什么，为什么要做这些改动。

### 2. 改动类型（多选）
- [ ] 新功能 (feature)
- [ ] Bug修复 (bug fix)
- [ ] 代码重构 (refactor)
- [ ] 性能优化 (performance)
- [ ] 文档更新 (documentation)
- [ ] 样式更新 (style)
- [ ] 测试相关 (test)
- [ ] 构建工具 (chore)

### 3. 测试 Checklist
- [ ] 单元测试通过
- [ ] 集成测试通过
- [ ] 手动测试通过
- [ ] 代码 Review 完成
- [ ] 文档已更新

### 4. 注意事项
{如果有需要 Reviewer 特别注意的地方，如：
- 破坏性变更
- 数据库迁移
- 配置变更
- 依赖更新
}

### 5. 截图/录屏
{如果是UI改动，提醒我补充截图或录屏}

### 6. 相关文档
{相关文档链接或说明}

## 输出要求

- 描述要详细，但不要废话
- 让 Reviewer 快速了解这个 PR
- 使用清晰的 Markdown 格式
- 包含所有必要的信息
- 保持专业和友好的语气

---

## 模板说明

好的 PR 描述能显著提高代码审查效率。此模板确保 PR 包含所有必要信息，帮助团队高效协作。

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `{简要说明PR内容}` | PR 做了什么、解决什么问题 | `添加了用户头像上传功能` |
| `{贴issue链接或编号}` | 关联的 Issue | `#123` |
| `{通过/未通过/未写}` | 各类测试状态 | `通过` |
| `{列出测了哪些功能}` | 手动测试场景 | `登录、注册、头像上传` |

### 适用场景

- PR 描述不知道怎么写
- 团队统一 PR 格式
- 确保 PR 包含完整的测试信息
- 便于 Reviewer 快速了解改动

### 使用建议

1. 简要说明改动内容，AI 会扩展为完整描述
2. 标注关联 Issue 便于追溯
3. 如实填写测试情况，帮助 Reviewer 评估风险
4. UI 改动记得补充截图或录屏

## 审查重点

请从以下角度审查代码：

### 1. 代码质量
- 命名是否清晰、一致
- 函数/方法是否单一职责
- 代码复杂度是否合理
- 注释是否恰当、有用

### 2. 功能正确性
- 边界条件处理是否完整
- 错误处理是否恰当
- 业务逻辑是否正确
- 是否有潜在bug

### 3. 性能考虑
- 是否有性能瓶颈
- 算法复杂度是否合理
- 内存使用是否高效
- 是否有不必要的计算

### 4. 安全性
- 是否有安全漏洞
- 输入验证是否充分
- 敏感信息处理是否安全
- 权限检查是否完整

### 5. 可维护性
- 代码结构是否清晰
- 是否遵循项目规范
- 测试是否容易编写
- 文档是否完整

### 6. 兼容性
- 浏览器/环境兼容性
- API兼容性
- 向后兼容性考虑

## 输出要求

1. **问题分类**：将发现的问题按严重程度分类（严重、中等、轻微）
2. **具体建议**：每个问题提供具体的改进建议和代码示例
3. **优先级排序**：建议修复的优先级顺序
4. **正面反馈**：指出代码中的优点和良好实践
5. **总结建议**：整体改进建议和学习要点

## 模板说明

此模板用于系统化代码审查，帮助发现代码质量问题，提升代码质量和团队技能。审查时应保持建设性态度，既指出问题也肯定优点。
